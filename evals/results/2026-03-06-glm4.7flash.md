## Test 8: Top 5 CPU Processes ⚠️

**Task**: `show me the top 5 processes using the most CPU right now`

### Cadmus (0.3s, ❌ FALSE POSITIVE)
```
✓ ▸ [NL] Plan: list — 1 step(s) operating on .:
  list dir

  (define (list (path : (Dir Bytes)))
    (bind path ".")
    (list_dir))

  → (app.js index.html system_report.txt todo.css todo.html todo.js)

✓ Completed in 1 tool call(s), 0.3s
```

**This is the worst failure mode.** The NL pipeline matched "show" → `list_dir`
(enumerate synonym), returned a file listing with a green ✓, and never reached
the LLM. The user asked for processes and got file names. The agent reported
success.

Root cause: "show" maps to the "enumerate" action in `nl_vocab.yaml`, which
maps to `list_dir`. The NL pipeline has no semantic check that the generated
plan matches the user's intent — it only checks syntactic/type validity.

### pi (23.4s, ✅)
```
1. Ollama runner - 17.8%
2. Druva inSync - 17.3%
3. iTerm2 - 17.0%
4. pi - 12.2%
5. pi - 4.3%
```

Ran `ps aux --sort=-%cpu | head` and formatted the output.
